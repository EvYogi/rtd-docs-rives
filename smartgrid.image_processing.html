smartgrid.image_processing.compute_GHI.clear_sky_ghi(zenith, doy)¶

Computes the Global Horizontal Irradiance (GHI) under a clear sky, for a given zenith value, at a given day of year This model is trained on actual data, and parameters are accessible in the configuration file conf.json

Parameters:
  • zenith (float) – value of the zenith to compute
  • doy (float) – floating value of the day of year
Returns:

floating value of the GHI computed
Return type:

float
smartgrid.image_processing.compute_GHI.predict_GHI(img_infrared, img_infrared_15, img_visual, timestamp, clearsky, overcast, *args)¶

Predicts GHI

Given two consequent infrared images, the corresponding visual image, a timestamp, and the reference images clearky & overcats, computes the GHI for the models given in *args

Parameters:
  • img_infrared (np.array) – the infrared image at the time of the prediction
  • img_infrared_15 (np.array) – the infrared image 15 minutes before the prediction (for block matching)
  • img_visual (np.array) – the visual image at the time of the prediction
  • timestamp (str) – any form of string timestamp, interpretable by numpy
  • clearsky (np.array) – the clearsky reference image (month & hour of the timestamp)
  • overcast (np.array) – the overcast reference image (month & hour of the timestamp)
  • args – list of models
Returns:

predictions and data point used for prediction
Return type:

tuple of tuples

smartgrid.image_processing.satellite_image_processing module¶

smartgrid.image_processing.satellite_image_processing.remove_borders_with_convolution(img, borders)¶

Remove the points referenced by borders on image. Apply a 2D convolution on img with a 3*3 unitary patch Convolution is applied arbitrarly 10 times in order to prevent from infinite loop

Parameters:
  • img (bytes) – binary encoding of satellite image
  • borders (numpy.ndarray) – boolean array of borders coordinates, size (615, 845)
Returns:

image with borders removed, size of input image
Return type:

PIL.Image

smartgrid.image_processing.solar_angles_computation.solar_angles_computation(universal_time, longitude, latitude)¶

Computes the zenith matrix associated with longitude, latitude at a given universal time, reference NOAA Global Monitoring Division

WARNING : longitude value is to be converted in radians within the calculus, and azimuth is given as the complementary of 2 Pi

Parameters:

  • longitude (numpy.ndarray) – array of the considered longitude
  • latitude (numpy.ndarray) – array of the considered latitude
Returns:

tuple containing zenith and azimuth values in degrees
